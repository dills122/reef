## How It Fits Together

Two layers, one command path. The **core platform** owns real business behavior — order intake, hidden-book matching, executions, and the post-trade path toward settlement. The **simulation control plane** — scenarios, participant bots, seeded randomness, and the emerging Bot Arena — drives that same platform through the identical commands a manual user would issue. Simulation never writes to domain tables directly.

```text
order intake -> hidden-book matching engine -> executions & trades
             -> allocations -> confirmations -> settlement -> audit trail
```

## Write A Trading Bot

Bots are TypeScript classes extending `ReefBotV1`. The harness owns scheduling, validation, and risk checks — your bot just decides what to do each tick.

```ts
export default class ExampleBot extends ReefBotV1 {
  static override metadata = {
    name: "example-bot",
    publisher: "Example Publisher",
    email: "bot-author@example.com",
    version: "1.0.0",
    sdkVersion: "1.5.0",
    botApiVersion: "v1",
  } as const;

  override async onTick(ctx: BotContextV1) {
    const snapshot = await ctx.marketData.snapshot("AAPL");
    if (!snapshot.ok) return [];

    return [
      ctx.orders.placeLimit({
        instrumentId: "AAPL",
        side: "BUY",
        quantity: 10,
        limitPrice: snapshot.value.midPrice - 1,
      }),
    ];
  }
}
```

```bash

make dev-smoke    # verify the local stack end to end
make dev-sim      # drive a seeded scenario
```
